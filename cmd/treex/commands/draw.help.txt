Draw tree diagrams from .info file data without filesystem validation.

The draw command renders tree structures directly from .info file annotations,
without checking if the paths exist in the filesystem. This is useful for:
- Visualizing planned directory structures
- Creating diagrams of conceptual hierarchies
- Documenting structures that don't exist yet

USAGE:
  treex draw --info-file=<filename>

The draw command requires an info file to be specified with the --info-file flag.
It will render the tree structure based on the paths and annotations in the file.

EXAMPLES:
  # Draw a family tree from family.txt
  treex draw --info-file=family.txt
  
  # Draw a planned project structure
  treex draw --info-file=project-plan.txt
  
  # Draw with specific format
  treex draw --info-file=structure.txt --format=markdown
  
  # Draw with depth limit
  treex draw --info-file=large-structure.txt --depth=3

SAMPLE INFO FILE FORMAT:
  Dad: Father of the family
  Mom: Mother of the family
  kids/: Children directory
  kids/Sam: Little Sam
  kids/Alice: Older sister

OUTPUT FORMATS:
  The draw command supports all the same output formats as the show command:
  --format=color       Full color terminal output (default)
  --format=no-color    Plain text output
  --format=markdown    Markdown format
  
  Use 'treex formats' for a full list of available formats.

DIFFERENCES FROM SHOW COMMAND:
  - Does not validate that paths exist in filesystem
  - Does not show warnings for missing paths
  - Always shows all entries (equivalent to --show=all)
  - Requires --info-file flag (no default .info file lookup)
